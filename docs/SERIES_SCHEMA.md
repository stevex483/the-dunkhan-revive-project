# Series Schema

**Version:** 1.0

**Last Updated:** 2026-07-07


This document defines the standard structure used by every series page in the database.

All documented series must follow this structure.

---

# General Rules

Every series page should describe only verified information.

Unknown information should be marked as "Unknown".

Assumptions must never be presented as facts.

---

# 1. Overview

A short description of the series.

---

# 2. General Information

| Field | Description |
|-------|-------------|
| Original Title | Official title of the series |
| Creator(s) | Main creator(s) |
| First Release Date | Date of the first known episode (if known) |
| Years Active | Years during which the series was produced |
| Minecraft Version | Minecraft version (if known) |
| Genre | Survival, Roleplay, Adventure, etc. |
| Number of Episodes | Known number of episodes |
| Recovery Status | Current recovery status |

---

# 3. Description

A more detailed description of the series.

This section should explain:

- the concept
- the gameplay
- the story (if applicable)
- notable features

---

# 4. Recovery Status

Possible values:

- 🟢 Fully Recovered
- 🟡 Partially Recovered
- 🔴 Mostly Lost
- ⚫ Lost

Include a short explanation of the current status.

---

# 5. Episode List

List every known episode in chronological order.

Each episode should link to its own dedicated page when available.

---

# 6. Known Playlists

List every known playlist related to the series.

---

# 7. Known Reuploads

Document community reuploads.

Include links whenever possible.

---

# 8. Related Investigations

List investigations related to this series.

Example:

- INV-0001 – Wayback Machine Archive Search

---

# 9. Sources

List all sources used to build the page.

Whenever possible, original sources should be preferred over secondary sources.

---

# 10. Notes

Additional information that does not fit into the previous sections.
