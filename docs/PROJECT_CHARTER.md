# Project Charter

This document defines the official standards of **The Dunkhan Revive Project**.

Every contributor is encouraged to follow these guidelines to ensure that the project remains consistent, transparent and scientifically rigorous over time.

# Core Principles

## 1. Preserve before everything

The primary objective of this project is the preservation of lost media.

Every recovered file, document or piece of evidence should be preserved.

---

## 2. Facts over assumptions

Every statement should be based on verifiable evidence.

Clearly distinguish:

- Facts
- Observations
- Hypotheses
- Personal memories

Never present a hypothesis as a confirmed fact.

---

## 3. Document every investigation

Every investigation should be documented.

Even unsuccessful investigations are valuable because they prevent duplicate work.

---

## 4. Preserve history

Completed investigations should never be rewritten.

If new evidence appears later, create a new investigation instead of modifying the old one.

The history of the project must remain preserved.

---

## 5. Cite original sources

Whenever possible:

- cite the original source
- preserve original URLs
- preserve screenshots
- preserve archived pages

---

## 6. Respect copyright

This project aims to preserve and document lost media.

The copyright of recovered videos always remains with their original creators.

---

## 7. Preserve original data

Never modify original evidence.

Whenever possible, preserve:

- original filenames
- original URLs
- original metadata
- original timestamps

Always keep an untouched copy of original material.

---

## 8. Transparency

If something is unknown, state that it is unknown.

Never guess.

Never present speculation as evidence.

Uncertainty is part of research.

---

# Investigation Rules

Every investigation must include:

- Investigation ID
- Date
- Objective
- Method
- Findings
- Conclusion
- Evidence
- Future Work
- Status

---

# Investigation Numbering

Investigations are numbered sequentially.

Examples:

- INV-0001
- INV-0002
- INV-0003

Numbers are never reused.

---

# Repository Philosophy

The repository should remain:

- simple
- well documented
- transparent
- easy to understand

Avoid unnecessary files or directories.

Every file should have a clear purpose.

---


# Motto

> Preserve the past. Restore the memory.

Recover, preserve and document the lost history of Dunkhan & SuperMonkeyMan for future generations.

---

# Source Reliability

Not every source has the same reliability.

The project distinguishes several levels of source reliability:

- Primary sources (original files, official archives)
- Secondary sources (trusted archives, mirrors)
- Community memories (useful but unverifiable)
- Hypotheses (must always be identified as such)

Whenever possible, primary sources should be preferred.

---

# Commit Messages

Commit messages should be:

- short
- descriptive
- written in English

Examples:

Create investigation INV-0001

Add HunterZ metadata

Update README

Document Ghostarchive research

Commit messages should describe what was actually changed.

Avoid vague commit messages such as:

- Update
- Fix
- Changes

---

# Long-Term Vision

The Dunkhan Revive Project is intended to become a permanent archive documenting the history of Dunkhan & SuperMonkeyMan.

The goal is not only to recover lost videos, but also to preserve their history, metadata, and cultural significance for future generations.
